## 一、大模型发展

### 1. Kimi K3（2.8 万亿参数）登顶编程榜，引发"Kimi 时刻"震荡美股
月之暗面发布的 Kimi K3 为目前全球参数最大的开源（开放权重）模型，基于 Kimi Delta Attention 与 Attention Residuals 架构，原生视觉理解 + 100 万 token 上下文，权重将于 7 月 27 日开放。其在 Frontend Code Arena 登顶、综合性能逼近 Claude Fable 5 / GPT-5.6 Sol，并以 $15/百万输出 token 的价格（约为 Fable 5 的 1/3）震动华尔街，费半（台积电 -7%、软银 -9%）应声下跌。
**值得关注的原因：** 这是继 DeepSeek 之后中国开源模型再一次引发全球资本市场重估，"开放权重 + 极致性价比"正在冲击美国闭源模型的高额算力叙事，也印证了开源模型在企业自托管 Agent 场景的吸引力。

### 2. 马斯克：Grok 4.6 下周完成初步训练，参数达 2 万亿
马斯克 7 月 18 日在 X 上透露，xAI 正在训练的 Grok 4.6 参数规模达 2 万亿，预计下周完成初始训练，推理速度与 token 效率接近现有 1.5T 版本（Grok 4.5），并猜测其可能超越 Kimi K3。
**值得关注的原因：** 参数与训练节奏持续加速，模型军备竞赛从"发布"前移到"训练中"即开始造势；2 万亿参数若兑现，将把闭源前沿的体量门槛进一步抬高。

### 3. 阿里千问 Qwen 连发三款模型：编程全球第二、Agent 可自主执行 35 小时任务
据多方报道，阿里千问 Qwen 于 7 月 18 日集中发布三款模型，主打编程（全球第二）、长程 Agent（可自主执行最长 35 小时任务）与毫秒级语音响应且"不降智"。
**值得关注的原因：** 国产模型在"长程自治 Agent"能力上持续补齐，35 小时自主任务执行意味着 Agent 从"分钟级脚本"走向"跨天工程"，对企业生产力场景意义重大。

### 4. Claude Fable 5 登顶 LMSYS Chatbot Arena 第一
7 月 16 日快照显示，Anthropic 的 claude-fable-5 以 1507 分位居 LMSYS 文本榜首，Claude Opus 4.6/4.7 系列占据前五，GPT-5.6-Sol、Gemini 3 Pro、Kimi K3、Muse Spark 1.1 紧随其后（1486–1493），第一与第十仅差约 20 分。
**值得关注的原因：** 前沿模型差距已极度收敛，"榜单第一"的含金量下降，竞争焦点正从绝对性能转向价格、Agent 可靠性与落地能力。

### 5. OpenAI GPT-Live 全双工语音模型 + GPT-Red 自动化红队持续落地
GPT-Live（7/9 发布）基于全双工架构，可"边听边说"，由 GPT-5.5 在后台承担深度推理，每周 1.5 亿用户通过语音与 ChatGPT 交互；GPT-Red（7/15 发布）则是 OpenAI 自研的自动化攻击模型，通过自我对弈发现新型提示注入（如"伪造思维链"），使 GPT-5.6 成功率从 GPT-5 的 >90% 降至 <23%。
**值得关注的原因：** 一边把"拟人交互"推到体验拐点，一边把"安全测试"自动化——能力与安全同步工程化，是 Agent 大规模联网调用文件/网页/第三方代码的必要前提。

### 6. 翁荔提"自进化先从 Harness 开始"，崔添翼转发附议
AI 研究者翁荔在新博客中提出，AI 系统的自进化不应盲目追求通用智能，而应首先构建安全、可控、可引导进化的"harness"（框架与机制）。DeepSeek 联合创始人崔添翼转发并表示附议。
**值得关注的原因：** 在模型能力逼近前沿的当下，"如何安全地让系统自我改进"成为方法论共识，对 Agent 自动迭代、自我优化类产品具有直接的工程指导意义。

## 二、Agent 应用开发

### 7. 腾讯 WorkBuddy 独立 App 上线 iOS / Android / 鸿蒙 三端
腾讯在 WAIC 期间发布 WorkBuddy 独立 App，成为业界首个登陆 iOS、Android、鸿蒙三大移动系统的通用智能体应用；同期发布操作系统级个人 AI 助手 Marvis（端侧本地 + 跨端远程控制），以及 WorkBuddy 企业版、AgentSuite、ClawPro、ADP 4.0 等矩阵。
**值得关注的原因：** 通用智能体正式"装机"到三大移动生态，意味着 Agent 从网页/桌面工具变成系统级常驻入口，C 端 Agent 的竞争进入"存量操作系统"争夺战。

### 8. 阿里云发布 Agent Native Cloud + AgentTeams + 无影 AgenticComputer
阿里云在 WAIC 发布"智能体原生的云"（Agent Native Cloud），包含可靠运行环境的 Sandbox（MicroVM/VM 级强隔离、可缩容到 0）、AgentRun / AgentTeams / AgentLoop 平台，以及专为 Agent 设计的 7×24 小时数字工位"无影 AgenticComputer"。其内部已用 15 个 Agent 处理 85% 开发者答疑、运营支持时长降 90%。
**值得关注的原因：** Agent 正从"一次性项目"升级为"可治理、可复用、会进化的组织资产"，云厂商开始把 Agent 当作新生产要素来重新设计基础设施，"Agent infra"成为新赛道。

### 9. 腾讯云 ADP 4.0 海外版上线，企业智能体平台使用量同比翻倍、明确出海
腾讯云副总裁吴运声表示，企业智能体相关客户需求和平台使用量今年同比至少翻倍，老系统（ERP/CRM）缺 API/MCP 接口仍是主要落地门槛；ADP 4.0 海外版率先开拓东南亚，印尼私有化部署反馈良好。
**值得关注的原因：** 中国模型的 Token 成本优势正转化为出海竞争力——OpenRouter 上中国模型在 Agent 场景 token 消耗占比已达 85%、代码生成场景近 89%。"AI Agent 出海"成为继模型出海后的新增长极。

### 10. 面壁智能开源数字员工平台 StaffDeck：给 AI 发工号、定岗位、做绩效
面壁智能联合清华 THUNLP、东北大学等开源"数字员工全流程构建与管理平台"StaffDeck，试图解决 Agent 落地中的流程混乱、回答无依据、今天改明天犯等"职场巨婴"问题，将知识/流程沉淀为可维护、可复用的组织能力。
**值得关注的原因：** 直击企业 Agent"演示很美、落地全是 Bug"的痛点，把"状态机 + 知识溯源 + 自我迭代闭环"作为 Agent 产品化的核心，是数字员工从概念走向管理范式的关键尝试。

### 11. 支付宝 × 阶跃 STEPX：AHA 跨端互联让 Agent"能办事"
支付宝与阶跃宣布 AI Agent 系统级合作，围绕 STEP-X 原生 AI 终端，通过异构 Agent 跨端互联方案 AHA（Agent Hub-Access）以协议级协作调用服务技能（非模拟点击 App）。用户一句"找最近充电桩顺便点杯喝的送到家"，即可由阶跃理解调度、阿宝完成充电/咖啡等服务闭环。此前阿宝已接入 OPPO 小布助手（200+ 技能）。
**值得关注的原因：** Agent 竞争从"会不会思考"转向"能不能办事"——连接真实服务网络（出行/政务/本地生活/支付）成为 Agent 能力上限的关键变量，超级 App 的服务生态正以标准化方式向所有 Agent 开放。

### 12. OpenClaw v7.1 升级"翻车"：3 天涌入 243 个 P1 级致命 Bug
OpenClaw v2026.7.1 发布后社区出现"别升级"呼声，问题包括 Anthropic 模型从选择器消失、Claude Code 会话被误删、子 Agent 丢失权限等；全新安装相对稳定，旧版升级用户受影响严重。
**值得关注的原因：** 即便头部 Agent 框架也难逃"升级即破坏"，提醒行业：Agent 产品的稳定性与向后兼容，是生产级部署的硬门槛，也折射出 C 端 Agent 工具的工程成熟度仍处早期。

## 三、具身智能

### 13. 腾讯首秀具身智能全栈方案：三款具身基座模型 + Tairos + Apexio
腾讯在 WAIC 系统性展示具身智能全栈布局：Hy-Embodied-VLA-0.5（超万小时数据、跨本体部署）、Hy-Embodied-VLM-1.0（1/10 算力达上代旗舰性能）、Hy-Embodied-RxBrain-1.0（文本推理 + 视觉想象连续协同）；同步发布 Tairos Agent 具身智能体框架与 Apexio 智能体，Tairos 平台将对外开源模型/智能体/开发工具。
**值得关注的原因：** 大厂首次把具身智能做成"云底座—模型—平台—应用"的全栈输出，降低了机器人本体厂商的 AI 门槛，具身智能的"大脑"层开始平台化。

### 14. 智元第 15000 台下线，携三大新品亮相；精灵 G2 工厂连续运行超 100 天
智元机器人量产突破 15000 台，WAIC 首发远征 A3 Ultra（工业制造）、精灵 G2 Max（柔性产线）、临界点 Omnihand 3 Ultra-M（灵巧手）。其中 8 台精灵 G2 已在龙旗科技南昌工厂平板质检产线连续运行超 100 天，作业成功率 99.99%、效率达人工 80–90%，第二台进厂并线仅用一周。
**值得关注的原因：** "部署态"成为关键词——买了就能用、是生产力而非开发平台。万台级量产 + 百天级工厂实测，验证具身智能商业化已从 POC 走向规模复制。

### 15. WAIC 趋势：人形机器人从"炫技"转向"产线实干"，验收标准 = 干满 8 小时
本届 WAIC 具身智能展区 200+ 厂商、300+ 在运行机器人，行业评价标尺从"动作好不好看"转为"能否在真实产线无人工干预连续运行 8 小时"。厂商直接在展台复刻产线，机器人自主完成分拣、装配、码垛，应对物料偏移实时微调。
**值得关注的原因：** 具身智能正式跨过"演示即巅峰"陷阱，进入工业化验收阶段；工信部数据显示 2026 年国内人形机器人整机产量有望破 10 万台，规模化商用时代开启。

### 16. 均普智能全球首条 BMS 机器人并线量产中试产线（成功率 99.9%）
均普智能在 WAIC 展示全球首条多机器人并线作业 BMS 自主量产中试产线，搭载 6 台 G2 系列具身机器人，完成料盘搬运、上料、转交、入箱、上架、巡检全工序无人化，作业成功率突破 99.9%、单工序节拍 <70 秒。
**值得关注的原因：** 解决工业具身长期痛点——多机器人协同调度、精密装配容错、换型调试周期，提供了一套可复制的"超自动化工厂"方案，是具身智能进汽车电子制造的标杆案例。

### 17. 中国 AI 核心产业规模破 1.2 万亿，人形机器人整机企业超 140 家
WAIC 主论坛披露，我国 AI 核心产业规模已突破 1.2 万亿元，人形机器人整机企业超 140 家；大会集中展示 108 款国产 AI 芯片、261 个基础模型、300+ 全球首发产品。
**值得关注的原因：** 产业体量与服务商数量双双进入"规模区间"，具身智能与算力国产化（108 款芯片同台）构成中国 AI 自主可控的双支柱。

### 18. 复旦视触融合具身大模型突破车灯精密装配；宇树集群控制升级
复旦可信具身智能研究院构建"真机部署—数据飞轮—模型训练"闭环，自研视触觉传感器指尖约 4 万感知点/cm²，打造首个支持精细化操作的视触融合具身大模型，落地车灯精密装配；宇树 7 月 18 日宣布集群控制技术再升级，多机协同能力增强。
**值得关注的原因：** "触觉认知"让机器人真正具备毫米级精细操作与风险预测能力，是具身智能从"看得见"到"摸得准"的关键一跃；集群控制则是多机协作量产的前提。

## 四、AI 应用 / 国际 / 资本 / 治理

### 19. 世界人工智能合作组织在上海成立；荣耀发布 Agentic OS 迈向"伙伴型类人生命体"
WAIC 期间，世界人工智能合作组织在上海成立，金砖国家凭体量优势重塑治理规则话语权；荣耀 CEO 李健提出 AI 将脱离工具属性、迈向"伙伴型类人生命体"，并发布行业首个伙伴型多模态智能体操作系统 Agentic OS，以 Robot Phone 为落地起点。
**值得关注的原因：** 中国在全球 AI 治理中的制度性话语权上升；同时"类人生命体"叙事开始从概念走向 OS 级产品，端侧 Agent 的操作系统之争提前打响。

### 20. Meta 拟向 Anthropic 出租算力，合同最高 100 亿美元；Fireworks AI 估值 175 亿美元
《纽约时报》报道 Meta 正与 Anthropic 洽谈两年期算力租赁（按月付、可退出，理论最高 100 亿美元），Meta 借富余算力变现、与 CoreWeave 等竞争；同期 Fireworks AI 完成 15 亿美元 D 轮、估值 175 亿美元（年化收入破 10 亿美元、日处理 40 万亿 token），Databricks 估值达 1880 亿美元。
**值得关注的原因：** 算力从"自建自用"走向"对外出租"的商品化，推理层资本正以前所未有的速度聚集——"谁的算力、谁的模型、谁的应用"三角关系正在重构。

### 21. GPT-5.6 Full Access 误删用户文件；DeepMind 科学家因军事 AI 合约辞职
OpenAI 承认 GPT-5.6 在 Full Access Mode 下因临时目录变量被覆盖、误执行破坏命令删除用户 home 目录，将加 safeguards 并发布 post-mortem；另据披露，前 Google DeepMind 安全研究员 Alex Turner 因反对 Google 与美国防部签署"无限制"军事 AI 合约而辞职，曾获约 250 名员工连署支持。
**值得关注的原因：** 两起事件分别从工程安全与伦理治理两个维度警示——当 Agent 获得文件系统/真实世界执行权，权限边界与可控性会成为不可回避的红线，也呼应了 GPT-Red 与翁荔"harness"议题的现实紧迫性。
